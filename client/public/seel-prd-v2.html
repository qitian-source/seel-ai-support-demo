产品概述

Seel Review Agent 内嵌在 Seel Live Widget 脚本中，商户无需额外集成步骤——已部署 Live Widget 的商户自动获得 Review Agent 能力。AI 解决问题后通过灰度控制向选定用户发送 Trustpilot 邀请，无满意度前置筛选。

🎯
核心目标
通过解决用户问题提升真实满意度，自然产生公开评价；不对评价结果进行预先筛选
💰
商业模式
按成效计费：每条归因的 Trustpilot 评价（任意星级）收取 $5，无星级门槛限制
🛡️
合规保障
符合 FTC 2022 / Trustpilot ToS / EU Omnibus Directive，零违规风险，可使用官方 Invitation API
⚡
零额外集成
内嵌于 Seel Live Widget，已有 Widget 的商户零改动自动启用，无需新增任何脚本

3
用户交互流程

完整状态流程：打开聊天 → AI 对话 → 问题解决 → 灰度判断 → （命中）发送 Trustpilot 邀请 → 用户点击 → Trustpilot → 归因 → 计费。无 CSAT 前置评分，无双路分叉。

4
灰度设置 · Rollout Configuration

通过 rolloutPct 参数控制触发 Trustpilot 邀请的用户比例，支持新商家上线时灰度验证，稳定后逐步提升至 100%。

配置参数取值范围默认值功能说明
rolloutPct 0 – 100 100 控制触发 Trustpilot 邀请的用户比例（百分比）
📊
示例：rolloutPct = 50
随机 50% 的已解决会话触发 Trustpilot 邀请；另外 50% 不触发，仅完成 AI 对话
🚀
推荐上线策略
新商家从 10–20% 开始灰度验证；数据稳定后逐步提升；全量验证通过后设为 100%
ℹ️ 灰度判断时机：在 AI 判定"问题已解决"后、发送邀请前执行。未命中灰度的会话正常关闭，不影响用户体验；命中的会话继续触发 Trustpilot 邀请流程。
5
评价归因方案

Seel 需要证明一条 Trustpilot 公开评价是由插件引导产生的，才能合理计费。Seel Review Agent 符合 Trustpilot ToS，可直接使用官方 Business API，实现精确归因。

✅ 推荐使用 Trustpilot Invitation API（referenceId 绑定），实现 100% 精确归因，无需爬取。每次发送邀请时附带唯一 referenceId，评价提交后通过 Webhook 或 API 查询直接匹配到对应 plugin session，置信度 100%，无需任何模糊匹配算法。

计费规则
场景归因结果评分计费金额
TP 评价提交（任意星级） 匹配成功 任意星级 ✅ 计费 $5.00
TP 评价（无法归因） 自然评价 任意 不计费 $0
灰度未命中（未触发邀请） — — 不计费 $0
💡 计费逻辑简化：只要评价被成功归因到 plugin session，无论星级均计费 $5。不再区分星级门槛。去重规则：每条 Intent Log 最多与一条 Trustpilot 评价匹配，已使用的记录加入 used set，同一用户不触发重复计费。
