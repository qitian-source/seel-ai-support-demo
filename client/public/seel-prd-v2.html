产品概述

Seel Review Agent（合规版）内嵌在 Seel Live Widget 脚本中，商户无需额外集成步骤——已部署 Live Widget 的商户自动获得 Review Agent 能力。方案 B 移除了满意度前置筛选，AI 解决问题后向全量用户同时展示 Trustpilot 邀请与私密反馈入口。

🎯
核心目标
通过解决用户问题提升真实满意度，自然产生公开评价；不对评价结果进行预先筛选
💰
商业模式
按成效计费：每条归因的 Trustpilot 评价（任意星级）收取 $5，无星级门槛限制
🛡️
合规保障
符合 FTC 2022 / Trustpilot ToS / EU Omnibus Directive，零违规风险，可使用官方 Invitation API
⚡
零额外集成
内嵌于 Seel Live Widget，已有 Widget 的商户零改动自动启用，无需新增任何脚本

4
双路并行设计

与方案 A 的核心区别：不做满意度前置筛选，问题解决后同时展示两个独立入口。两条路径互不依赖，用户拥有完全自主选择权。

🌟
Trustpilot 公开评价
1. AI 解决问题后自动展示邀请
2. 用户自主选择星级（1–5 星）
3. 点击跳转 Trustpilot
4. 评价提交后归因并计费 $5
所有用户均可见，含差评用户
📝
私密反馈（可选）
1. 与 TP 邀请同时展示
2. 用户自主决定是否填写
3. 提交后仅内部可见
4. 不发布到任何公开平台
不替代 TP 邀请，互不影响
✅ 设计原则：两个入口并列呈现，互不依赖。用户可选择仅做 TP 评价、仅提交私密反馈、两者都做，或都不做。系统不以私密反馈替代或屏蔽 Trustpilot 入口。
🔄 方案对比 — 方案 A vs 方案 B
方案 A（CSAT 前置筛选）
  • 先展示满意度评分（😊 / 😐 / ☹️）
  • 仅 Satisfied 用户可见 TP 邀请
  • Neutral / Unhappy 路径不触达 TP
  • 计费门槛：仅 5 星评价
  • ⚠️ 违反 FTC / Trustpilot ToS
  • ⚠️ 不可使用官方 Invitation API
方案 B（全量并列，本文档）
  • 无前置筛选，直接展示双入口
  • 所有用户均可见 TP 邀请
  • 私密反馈与 TP 邀请并列独立
  • 计费门槛：任意星级均可
  • ✅ 符合 FTC / Trustpilot ToS
  • ✅ 可使用官方 Invitation API
5
评价归因方案

Seel 需要证明一条 Trustpilot 公开评价是由插件引导产生的，才能合理计费。方案 B 因符合 Trustpilot ToS，可直接使用官方 Business API，实现精确归因。

✅ 方案 B 推荐使用 Trustpilot Invitation API（referenceId 绑定），实现 100% 精确归因，无需爬取。每次发送邀请时附带唯一 referenceId，评价提交后通过 Webhook 或 API 查询直接匹配到对应 plugin session，置信度 100%，无需任何模糊匹配算法。
6
计费规则
场景归因结果评分计费金额
TP 评价提交（任意星级） 匹配成功 任意星级 ✅ 计费 $5.00
TP 评价（无法归因） 自然评价 任意 不计费 $0
私密反馈提交 — — 不计费 $0 （服务改进价值）
💡 计费逻辑简化：只要评价被成功归因到 plugin session，无论星级均计费 $5。不再区分星级门槛。去重规则：每条 Intent Log 最多与一条 Trustpilot 评价匹配，已使用的记录加入 used set，同一用户不触发重复计费。
