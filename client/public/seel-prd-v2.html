产品概述

Seel Review Agent 内嵌在 Seel Live Widget 脚本中，商户无需额外集成步骤——已部署 Live Widget 的商户自动获得 Review Agent 能力。AI 解决问题后通过灰度控制向选定用户发送 Trustpilot 邀请，无满意度前置筛选。

🎯
核心目标
通过解决用户问题提升真实满意度，自然产生公开评价；不对评价结果进行预先筛选
💰
商业模式
按成效计费：仅五星（5★）归因评价收取 $5；1–4 星归因评价不计费
🛡️
合规保障
符合 FTC 2022 / Trustpilot ToS / EU Omnibus Directive，零违规风险，可使用官方 Invitation API
⚡
零额外集成
内嵌于 Seel Live Widget，已有 Widget 的商户零改动自动启用，无需新增任何脚本

用户交互流程

完整状态流程：打开聊天 → AI 对话 → 问题解决 → 灰度判断 → （命中）发送 Trustpilot 邀请 → 用户点击 → Trustpilot → 归因 → 计费。无 CSAT 前置评分，无双路分叉。

4
邀请量上限 · Daily Invite Limit

通过 dailyInviteLimit 参数设置每日最多向多少用户发送 Trustpilot 邀请。新商家可从小量开始验证效果，稳定后逐步提升或设为无限制。

配置参数取值范围默认值功能说明
dailyInviteLimit 1 – ∞（0 = 无限制） ∞ 无限制 每自然日（UTC 00:00 重置）最多触发邀请的用户数量；当日已达上限的会话正常关闭、不发邀请
📊
示例：dailyInviteLimit = 50
每天最多向 50 名用户发送邀请；第 51 个符合条件的会话当日不触发邀请，次日零点重置计数
🚀
推荐上线策略
新商家从 10/天开始观察归因与评价质量；稳定后提升至 50–200/天；确认无异常后设为无限制
ℹ️ 判断时机：在 AI 判定"问题已解决"后、发送邀请前检查当日已发量。未达上限则正常触发邀请；已达上限的会话正常关闭，不影响用户体验。
5
评价归因方案

核心问题：Trustpilot 不向第三方提供 Webhook 或回调接口。用户点击邀请跳转至 Trustpilot 后，Seel 无法直接得知该用户是否真的提交了评价。

解决方案（模糊归因）：用户点击邀请按钮时，Seel 写入一条 Intent Log，记录本次"评价意图"。随后，系统主动爬取商家 Trustpilot 页面，将新出现的评价与 Intent Log 做多维度匹配，推断归因关系。置信度达标 + 五星 → 触发计费。

Intent Log · 记录字段（用户点击邀请按钮时写入）
字段说明
session_idSeel plugin session 唯一标识
merchant_id商家在 Trustpilot 上的主页标识（用于锁定爬取目标）
customer_name买家姓名（来自 Shopify 订单，用于后续评价人名字模糊匹配）
order_id订单号（用于评价正文关键词匹配）
invite_clicked_at用户点击邀请按钮的 UTC 时间戳，归因窗口从此刻起算
device_fingerprint浏览器/设备指纹（Canvas + UA + 分辨率哈希）
attribution_expires_atinvite_clicked_at + N天（N 为归因窗口设置，默认 T+3，可选 T+2 / T+3 / T+4 / T+5 / T+6），爬虫轮询的终止时间；到期后 log 关闭，不再参与匹配
爬取时间参数（商家后台可配置）
参数默认值可选范围含义
trigger_delay 20 min 5 – 60 min Intent Log 写入后，延迟多久触发第一次 TP 爬取。给用户足够时间在 Trustpilot 完成提交。
crawl_interval 1 天 固定 1 天 首次爬取后，每天爬取一次，固定节点：T+2 / T+3 / T+4 / T+5 / T+6（天）。
attribution_window T+3 T+2 / T+3 / T+4 / T+5 / T+6 归因窗口截止天数（T = 用户点击邀请当天）。窗口到期后 log 自动关闭，不再参与匹配。
⚡ 归因触发流程 · Intent Log 监测 + 周期爬虫匹配

1
用户点击邀请（T） → 写入 Intent Log，归因窗口开启
Widget 检测到"⭐ 在 Trustpilot 留评"点击事件 → 立即将上表字段写入 Intent Log → 用户跳转至 Trustpilot。attribution_expires_at = T + N天（N 由商家在后台配置，默认 T+3）开始计时。
2
T+2 / T+3 / … / T+N：每天爬取一次
从 T+2 起，每天爬取商家 TP 主页增量评价（new_reviews），对所有仍开放的 Intent Log 运行多信号匹配算法。一旦某次命中，立即标记归因成功并关闭该 log，不再继续轮询。超过配置的截止天数（默认 T+3，最大 T+6）仍未命中则窗口关闭。
3
置信度达标 → 归因成功；五星 → 触发计费
每轮匹配中，置信度 ≥ 阈值（默认 70，可配置）→ 归因成功，写入计费候选。仅当星级 = 5★ 时收取 $5；1–4 星归因成功但不计费。超出归因窗口仍未匹配的 log 自动关闭，标记为"未归因"，不计费。
匹配信号说明
信号判断方式权重
时间窗口 评价发布时间在 invite_clicked_at（T）之后、attribution_expires_at（T+N 天，默认 T+3，可选 T+2 ～ T+6）之前；发布越早置信度越高 高
评价人姓名 Trustpilot 评价人显示名与 customer_name 做模糊匹配（Levenshtein 相似度 ≥ 0.8 视为匹配） 高
订单号关键词 评价正文中包含 order_id 或其部分片段（后 6 位），视为强匹配信号 中
设备指纹 与 device_fingerprint 一致的浏览器在 invite click 后访问了 Trustpilot（通过 Seel JS SDK 的跨页面指纹追踪记录） 中
点击-发布时间差 invite_click → review_submit 间隔越短（如 < 30 分钟），归因概率越高，对置信度额外加权 中
💡 去重规则：每条 Intent Log 最多与一条 Trustpilot 评价匹配（1:1），已匹配的 log 加入 used set，不再参与后续匹配。同一用户在同一 session 周期内不触发重复计费。超过归因窗口（默认 T+3，最大 T+6）仍未匹配的 log 条目自动关闭，不再计费。
