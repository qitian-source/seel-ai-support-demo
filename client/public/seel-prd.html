产品概述

Seel Review Agent 内嵌在 Seel Live Widget 脚本中，商户无需额外集成步骤——已部署 Live Widget 的商户自动获得 Review Agent 能力。AI 客服解决问题后，自动触发满意度评分流程，将正向体验导向 Trustpilot 公开评价。

🎯
核心目标
将客服满意度转化为真实公开评价；将负面体验拦截在私域，保护商户公开评分
💰
商业模式
按成效计费：每条被归因的 5 星 Trustpilot 评价收取 $5，其他评分及差评路径不计费
🛡️
风险防护
中性反馈（预估 $75）和差评（预估 $180）均转入私密通道，不触达 Trustpilot
⚡
零额外集成
内嵌于 Seel Live Widget，已有 Widget 的商户零改动自动启用，无需新增任何脚本

CSAT 三层路径详解

AI 客服对话结束后，系统自动检测问题已解决，触发 CSAT 评分卡。用户从三个选项中选择，进入对应处理通道。

😊
Satisfied — 公开好评路径
1. 展示 Trustpilot 评分邀请卡
2. 用户点击星级（1–5 星）
3. 新标签直接跳转 Trustpilot
4. 评价提交后 Seel 归因并计费 $5
😐
Neutral — 私密改进路径
1. 展示私密反馈表单
2. 用户填写改进建议
3. 提交后仅内部团队可见
4. 不触达 Trustpilot，不计费
5. 预估拦截风险价值 $75
☹️
Unhappy — 危机管理路径
1. 展示经理升级表单
2. 用户描述问题经过
3. 提交后专属经理 24h 跟进
4. 严格私密，绝不公开
5. 预估拦截风险价值 $180
⚠️ 设计原则：三条路径均设有返回按钮，用户可随时返回 CSAT 重新选择。Neutral 和 Unhappy 路径下，系统绝不出现任何 Trustpilot 链接或入口。
5
评价归因方案

Seel 需要证明一条 Trustpilot 公开评价是由插件引导产生的，才能合理计费。根据商户是否购买 Trustpilot Business 套餐，采用两种不同技术方案。

6.1  免费版 — 抓取 + 三重模糊匹配

无 API 权限，每日爬取商户 Trustpilot 公开主页，对照意图日志（Intent Log）进行多级推断匹配。

6.2  付费版对比 — Trustpilot Business API

购买 Trustpilot Business 的商户可通过 Invitation API 在 Live Chat 内直接展示带 referenceId 的追踪链接，实现 100% 精确归因，无需任何爬取或匹配算法。

✅ 产品分层机会：将 Trustpilot Business API 方案作为高价值商户的升级路径，主打「100% 置信度归因 + 实时 Webhook + 转化率漏斗追踪」，与免费版形成明确差异。
6
计费规则
场景归因结果评分计费金额
Satisfied → TP 评价提交 匹配成功 5 星 ✅ 计费 $5.00
Satisfied → TP 评价提交 匹配成功 1–4 星 不计费 $0
Satisfied → TP 评价（无法匹配） 自然评价 任意 不计费 $0
Neutral → 私密反馈 — — 不计费 $0 （风险拦截价值 $75）
Unhappy → 经理升级 — — 不计费 $0 （风险拦截价值 $180）
🔒 去重规则：每条 Intent Log 最多与一条 Trustpilot 评价匹配（先到先得），已使用的 Log 加入 used set，同一用户不会触发重复计费。
