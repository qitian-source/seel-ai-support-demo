本次产品更新说明
围绕「AI Manager = 数字员工管理中枢」的定位，对会话列表复用、设置中心、Onboarding 引导与左侧导航做了一轮一致性打磨。
版本Seel AI Support v2

影响文件7 个源文件
0概述

本轮更新不引入新功能模块，而是对既有信息架构做收敛与复用，核心目标有三：

  • 单一数据源：会话列表组件抽离，Performance 与 Live Widget 共用，避免逻辑重复。
  • 配置归位：把分散的设置统一进全局 Settings 中心，按「渠道 / 数字员工」两个维度分层。
  • 引导更顺：Onboarding 步骤的跳转目标与文案更贴合「先选渠道、政策可选」的真实路径。
本次更新共涉及 7 个前端源文件，无新增依赖；TypeScript 类型检查仅余 3 个历史遗留错误（与本次改动无关）。
1Live Widget 会话列表复用

原先会话日志表格只存在于 Performance → Conversations。商家在 Live Widget 渠道页下的 Conversations 子标签只能看到占位空页面,需要切换到 Performance 才能查看该渠道对话——动线割裂。

改造前

会话表格内嵌在 PerformancePage 中,约 265 行逻辑;Live Widget 的 Conversations 子标签是空占位。

改造后

抽离为独立组件 ConversationsView;Live Widget 与 Performance 共用同一组件,单一数据源。

关键设计:lockedChannel 入参
  • 组件接收可选的 lockedChannel?: ChannelValue。
  • Live Widget 传入 "Live Chat Widget" → 列表锁定该渠道,隐藏渠道下拉筛选,展示一枚不可移除的「Channel: Live Chat Widget」上下文标签。
  • Performance 不传该参数 → 保留完整的全渠道筛选能力,行为与改造前完全一致。
2Settings 中心重构

全局 Settings 由单一「跳板列表」升级为带子标签的设置中枢,把数字员工配置真正收纳进来,而不再只是把人甩到别处。

子标签承载内容入口
ChannelsLive Widget / Email Inbox / Zendesk 三个渠道的设置跳板,展示各自连接状态(Live / Training / Connected / Not connected)。默认子标签
Rep ConfigAI 数字员工配置(ConfigureAgentSection)内联托管:人设、权限、共享默认值。AI Manager 页面本身回归「纯操作」。深链跳转
深链机制:新增一次性意图标志 agentSettingsIntent。「编辑人设」或 Onboarding 的「Create an AI rep」会带着该标志进入 Settings,自动定位到 Rep Config 子标签后立即复位。
3Onboarding 引导优化
调整  Connect a channel 的跳转目标
改造前

直接深链进 Live Widget 渠道的设置子标签——默认就替商家选好了「Chat」渠道。

改造后

跳转到 Settings → Channels 渠道选择列表,由商家自己挑选要接入哪个渠道。

调整  Import Policies 文案
  • 步骤标题改为 「Import Policies (optional)」,把「可选」直接写进标题,任何状态下都可见。
  • 移除原先仅在未完成时才出现的独立「Optional」徽标,避免与标题语义重复。
  • 「Done」徽标在完成后照常显示;该步骤始终不计入「essentials」进度条。
4左侧导航调整

把左侧栏的 Conversation 从顶部主导航组移到 WORKFORCE 分组下,作为该分组的第一项(位于 Support agent / Sales agent / VOC agent / Review agent 之上)。

  • 顶部主导航组现在止于 Reviews。
  • Conversation 保留当前的 disabled「coming soon」占位行为不变。
  • 语义上更合理:对话隶属于「数字员工工作台」这一范畴。
