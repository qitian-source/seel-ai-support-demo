一、整体布局重构

对标 Gmail，重新组织搜索、过滤与三栏结构

旧版 Email Inbox 搜索框位置不合理，Compose 按钮不够显眼，筛选条件与列表混排，视觉层级混乱。

🔍 搜索栏置顶

  • 搜索框移至左侧列表顶部，始终可见；
  • 支持按主题、发件人、内容实时过滤工单列表；
  • 清空按钮（×）在有输入内容时显示。

✍️ Compose 固定按钮

  • 左上角新增 "Compose" 主操作按钮，紫色高亮，位置固定；
  • 点击弹出新建邮件草稿（发件人默认为当前坐席邮箱）。

🔽 筛选条件重组

  • 状态筛选（All / New / Open / Pending / Solved）以水平 Tab 形式展示于搜索栏下方；
  • 类型筛选（User / Non-user）与排序（Newest / Oldest）合并至右侧 Icon 按钮，点击展开 Dropdown，减少界面噪音。

📐 三栏布局比例优化

栏位内容比例
左栏工单列表26%
中栏消息详情（ThreadView）46%
右栏工单信息面板（TicketInfoPanel）28%

左右分栏支持拖拽调整宽度（ResizablePanel）。

📌 回复后列表位置保持不变

  • 旧版发送回复后，工单列表会自动滚回顶部，需要重新下滑找到当前工单；
  • 现版本发送回复后，列表保持原有滚动位置，工单不重新排序，不自动跳转。
🔒

二、Internal Notes（内部备注）

替代旧版 Labels，引入消息级与工单级两种备注类型

旧版 Labels 适合分类标记工单，但不适合坐席之间的协作沟通。本次参考 Zendesk 设计，拆分为两种独立的笔记类型。
类型作用域可见范围持久化方式
Internal Note 单条消息 仅坐席可见 随工单保存
Customer Notes 当前工单 仅坐席可见 随工单保存

2.1 Internal Note — 消息级内部备注

入口

回复栏左下角新增模式切换按钮，默认为"Public reply"，点击可切换至"Internal note"：

✉ Public reply ▾
→ 切换后 →
🔒 Internal note ▾

切换后：

  • 输入框背景变为淡琥珀色，视觉提示当前为内部模式；
  • 发送按钮变为 "Add note"，样式为琥珀色边框。

消息渲染样式

内部备注在消息流中以区别样式展示：

  • 气泡背景：琥珀黄（bg-amber-50 border-amber-200）；
  • 顶部标签：🔒 Internal Note · Only visible to agents；
  • 不触发工单状态变更（不会将工单置为 Open）。
设计说明：Internal Note 的核心价值在于"不打扰客户"——坐席可以在工单内部讨论处理思路、记录异常、@同事，这些内容对客户完全不可见。

2.2 Customer Notes — 工单级备注

位置

右侧 Ticket Info Panel 顶部，独立分区 "Customer Notes"，替换原有 Labels 分区。

交互行为

  • 多行文本输入框，支持自由输入；
  • 实时保存（onChange 即更新），无需手动确认；
  • 备注内容仅属于当前工单，切换其他工单后各自独立不共享。
设计说明：Customer Notes 用于记录坐席在处理本工单时的临时观察与注意事项（如沟通要点、特殊承诺等），仅对坐席可见，不影响客户侧展示。
🖼

三、图片附件预览与缩放（Image Lightbox）

参考 Gmail，全屏沉浸式图片查看体验

旧版附件仅提供文件名 + 下载链接，用户无法在当前页直接预览图片，需跳转到新标签页，影响处理效率。

3.1 附件缩略图

当消息包含图片附件（mimeType 为 image/*）时，在消息气泡底部展示缩略图网格：

  • 每张缩略图尺寸：110 × 80 px，object-cover 裁剪，圆角边框；
  • Hover 时显示半透明蒙层 + 放大镜图标（Maximize2）；
  • Hover 时底部渐变遮罩显示文件名；
  • 点击任意缩略图，打开 Lightbox 并定位到对应图片。

非图片附件（PDF、文档等）以文件卡片形式展示（文件名 + 大小 + 下载按钮）。

3.2 Lightbox — 图片灯箱

参考 Gmail 附件预览设计，全屏沉浸式查看体验。

布局结构（从上到下）

┌──────────────────────────────────────────────────┐
│ crash_site_photo.jpg                       [×] │  ← 顶部栏：文件名 + 关闭
├──────────────────────────────────────────────────┤
│                                                   │
│         [‹]        [ 图片主体 ]        [›]      │  ← 图片区：左右切换箭头
│                                                   │
├──────────────────────────────────────────────────┤
│               ╔════════════════╗              │
│               ║  ⊖ │ 100% │ ⊕  ║              │  ← 底部缩放工具条
│               ╚════════════════╝              │
└──────────────────────────────────────────────────┘

缩放控制

控件功能
⊖ 按钮缩小 25%，最小值 25%
100% 百分比显示当前缩放比例；点击重置为 100%
⊕ 按钮放大 25%，最大值 300%
鼠标滚轮向上放大，向下缩小

拖拽平移

  • 缩放倍率 > 1 时，按住鼠标左键可拖拽移动图片；
  • 切换图片或重置缩放后，偏移量自动归零。

多图导航

  • 左右箭头按钮（圆形深灰背景，Hover 变亮）；
  • 切换时自动重置缩放和偏移；
  • 仅有单张图片时，左右按钮仍显示（不隐藏，避免布局跳动）。

键盘快捷键

ESC关闭灯箱
←→上一张 / 下一张
+放大
-缩小
0重置缩放

关闭方式

  • 点击右上角 × 按钮；
  • 点击图片区域以外的背景蒙层；
  • 按 ESC。

四、分组视图（按发件人合并）

将同一发件人的多封工单折叠聚合，一键切换列表展示模式

当同一客户提交多封工单时，默认平铺视图需要反复上下滚动才能找到关联工单，效率低。分组视图将相同发件人的工单收拢为一个客户组，整体一目了然。

🔀 模式切换入口

工单列表过滤行右侧新增 👥 分组按钮，与排序/筛选 Dropdown 并排展示：

☰ 平铺视图
点击切换
👥 分组视图
  • 激活状态下按钮高亮（绿色边框），再次点击还原平铺；
  • 切换不影响当前选中工单，也不重置状态筛选 Tab 或搜索关键词。

📋 分组列表结构

启用分组视图后，工单列表按 customerEmail 聚合，每个客户形成一个可折叠的组：

客户组头部（Group Header）

  • 客户头像：取客户姓名首字母，圆形色块背景；
  • 客户姓名 + 邮箱地址（小字灰色）；
  • 工单数量 Badge：显示该客户下工单总数（如 2 tickets）；
  • 未读 Badge：若有未读工单，显示红色数字角标；
  • 点击组头部可折叠 / 展开子列表（默认展开）；
  • 折叠时显示一个箭头 ▶，展开时显示 ▼。

组内工单条目（Thread Item）

  • 与平铺视图保持一致的工单卡片样式（主题、最新消息摘要、时间、状态 chip）；
  • 左侧增加缩进（16 px），与组头部形成层级关系；
  • 点击条目后，中栏消息详情正常加载，右侧 Ticket Info Panel 正常显示。

🗂 排序规则

  • 各客户组按组内最新工单的时间排序（Newest 优先）；
  • 组内工单按自身时间独立排序；
  • 切换"Oldest"排序时，规则同步应用于组级和条目级排序。
Demo 数据：data.ts 中已内置 Michael Zhang（2 封）和 Sarah Okonkwo（2 封）的多工单数据，可直接在 Demo 中切换分组视图验证效果。
